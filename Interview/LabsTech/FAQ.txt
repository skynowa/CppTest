Высылаю фидбеки по предыдущим кандидатам (вдруг будет полезно и поможет)
Также если будут какие-то вопросы, то фил фри как говориться. Помогу чем смогу :)

- Functions declared and used in only one file should be declared as static. e.g. accept_clients, handle_client.
- Extern variables should be avoided as much as possible. If this can not be avoided, the extern declaration should be
  in a header (e.g. extern struct xserver sv). I m not 100% sold on the idea of declaring a variable in a file where
  it is not used (e.g. struct xserver sv).
- Do not comment out code. Only in Readme
- Macros should be defined in a header file.
- It is a best practice to write macros with capital letters.
- Macros that ‘return’ or ‘break’ should not be used. It makes the code hard to debug. It's “terible ideea”.
  e.g. checkerr, checkforzero, do_compress, checklen
- Keep coding style consistent. Keep static functions at the bottom and put prototypes at the beginning like
  you did in loop.c
